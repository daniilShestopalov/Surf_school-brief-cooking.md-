## 3. РЕЕСТР НЕСООТВЕТСТВИЙ И ОТКЛОНЕНИЙ (Gaps & Discrepancies)

1. **[Major] Заглушка в логике сохранения аллергий (BS-002)** - ИСПРАВЛЕНО
   - *Расположение:* `development/client/features/booking/ui/AllergiesSelectionBottomSheet.kt`
   - *Описание:* В Compose-разметке на кнопке подтверждения оставлен комментарий `onClick = { /* TODO: Save allergies intent */ }`. Интент не отправляется в `ScreenModel`.
   - *Решение:* Добавлен интент `SaveAllergies`, кнопка в UI теперь отправляет интент в `BookingScreenModel`.
   
2. **[Major] Заглушка в логике отмены бронирования (BS-003)** - ИСПРАВЛЕНО
   - *Расположение:* `development/client/features/booking/ui/CancellationBottomSheet.kt`
   - *Описание:* На кнопке отмены записи оставлен комментарий `onClick = { /* TODO: Implement cancellation intent */ }`. Логика отмены брони пользователем недоступна.
   - *Решение:* `CancellationBottomSheet` подключен к экрану деталей записи, добавлен интент отмены `CancelBooking`.

3. **[Minor] Отсутствует отображение Email на экране профиля (SCR-006)** - ИСПРАВЛЕНО
   - *Расположение:* `development/client/features/profile/ui/ProfileScreen.kt`
   - *Описание:* Доменная модель `Client` содержит поле `email`, однако на экране `ProfileContent` выводятся только имя, телефон и аллергии. Контактный Email упущен из UI.
   - *Решение:* Поле `email` добавлено в Compose-разметку.
