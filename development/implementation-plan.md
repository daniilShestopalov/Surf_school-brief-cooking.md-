# Implementation Roadmap & Checklist

## Этап 1: Инфраструктура и Core-слой (Foundation)
- [x] Настроить базовую многомодульную/пакетную структуру проекта (`core`, `domain`, `features`) согласно принципам Feature-Driven Development.
- [x] Интегрировать и настроить DI-контейнер (Koin) для инициализации базовых сервисов и платформозависимых компонентов.
- [x] Настроить сетевой клиент Ktor (Network Wrapper) с базовой конфигурацией и сериализацией (kotlinx.serialization).
- [x] Реализовать глобальный перехватчик (HttpResponseValidator) для автоматической обработки `401 Unauthorized` со сбросом сессии и редиректом.
- [x] Реализовать перехватчик для обработки `429 Too Many Requests`, парсинг заголовка `Retry-After` и маппинг в `RateLimitException`.
- [x] Создать и внедрить провайдер идемпотентности для генерации уникальных `Idempotency-Key` (UUIDv4) на слое Presentation.
- [x] Интегрировать Multiplatform Settings и настроить локальное безопасное хранилище (Secure Storage) для сессий и Bearer-токена.

## Этап 2: Доменный слой и Shared Components
- [x] Реализовать базовые абстракции MVI (интерфейсы `State`, `Intent`, `Effect` и базовый `ScreenModel`).
- [x] Описать data-классы доменных моделей (`Client`, `ClassProgram`, `Chef`, `Slot`, `Booking`) с использованием `Integer` для денежных полей (копейки) и `UUID` для идентификаторов.
- [x] Настроить базовую навигацию на основе Voyager для поддержки переходов между экранами (Screen) и вложенной BottomSheet-навигации.
- [x] Настроить базовую дизайн-систему (токены цветов, типографики, отступов) и общие UI-компоненты (кнопки, снекбары, диалоги).
- [x] Реализовать изолированный expect/actual UI-канал для отображения карт со студиями (shared component для экранов деталей слота и бронирования).

## Этап 3: Пофичевая реализация (Feature-by-Feature Checklist)

### 3.1 Фича: Авторизация (SCR-001, SCR-002)
- [x] Создать MVI-компонент (`ScreenModel`, `State`, `Intent`, `Effect`) для логики запроса OTP, ввода кода и регистрации, включая обработку `429 Too Many Requests`.
- [x] Сверстать UI-экраны ввода телефона и OTP (Default, Loading, Error states) на Compose.
- [x] Интегрировать UI авторизации со слоем логики, внедрить сохранение токена в Secure Storage и настроить навигацию на главный экран.

### 3.2 Фича: Каталог слотов и расписание (SCR-003, SCR-004)
- [x] Создать MVI-компонент для загрузки расписания, фильтрации по датам и получения детальной информации о слоте.
- [x] Сверстать UI-экраны расписания и деталей слота (включая отображение данных шефа, цены в рублях, загрузку изображений через Coil 3).
- [x] Реализовать продуктовые состояния для расписания: Empty state ("Нет доступных классов") и Loading state.
- [x] Интегрировать UI со слоем логики, обработать состояние `NetworkException` (Offline) с показом снекбара и ошибку `410 Gone` при попытке открыть удаленный слот.

### 3.3 Фича: Бронирование (BS-001, BS-002, BS-003, SCR-005)
- [x] Создать MVI-компонент для процесса бронирования: калькуляция итоговой стоимости, обработка флага аренды, генерация `Idempotency-Key`.
- [x] Реализовать обработку бизнес-ошибок при бронировании (409 SLOT_FULL, 409 EQUIPMENT_OUT, 410 Gone, Network Timeout) с соответствующими Side Effects (алерты, снекбары).
- [x] Сверстать UI шторки оформления брони (BS-001), экрана реквизитов с таймером оплаты (SCR-005), шторки отмены (BS-003) и выбора аллергий (BS-002).
- [x] Интегрировать UI со слоем логики бронирования и привязать к графу навигации.

### 3.4 Фича: Профиль и активные записи (SCR-006, SCR-007, BS-004)
- [x] Создать MVI-компонент для загрузки списка активных бронирований клиента и отправки оценки шефу.
- [x] Сверстать UI экрана профиля (SCR-006), деталей предстоящей записи (SCR-007) и шторки оценки шефа (BS-004).
- [x] Реализовать Empty state ("Нет активных записей") для списка бронирований.
- [x] Интегрировать UI со слоем логики профиля, обработать успешные и ошибочные сценарии (например, таймаут при отправке рейтинга, загрузке профиля).

## Этап 4: Интеграционное тестирование и стабилизация (E2E Readiness)
- [x] Настроить и провести сквозную проверку транзакционного сценария: Выбор слота -> Шторка подтверждения (BS-001).
- [x] Проверить передачу параметров `slot_id`, `needs_rental_equipment` и жестко заданного `seats_count = 1`.
- [x] Подтвердить корректную привязку `Idempotency-Key` к POST-запросу `/bookings` и его сохранение в стейте при сетевых ретраях.
- [x] Протестировать обработку серверных конфликтов: корректное отображение UI при 409 SLOT_FULL, 409 EQUIPMENT_OUT, 410 Gone и 429 Too Many Requests.

- [x] Реализовать сложную мок-логику Бронирования (успех, отлов Idempotency-Key, симуляция ошибок 409 SLOT_FULL и 409 EQUIPMENT_OUT).
