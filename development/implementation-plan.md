# Implementation Roadmap & Checklist

## Этап 1: Инфраструктура и Core-слой (Foundation)
- [ ] Настроить базовую многомодульную/пакетную структуру проекта (`core`, `domain`, `features`) согласно принципам Feature-Driven Development.
- [ ] Интегрировать и настроить DI-контейнер (Koin) для инициализации базовых сервисов и платформозависимых компонентов.
- [ ] Настроить сетевой клиент Ktor (Network Wrapper) с базовой конфигурацией и сериализацией (kotlinx.serialization).
- [ ] Реализовать глобальный перехватчик (HttpResponseValidator) для автоматической обработки `401 Unauthorized` со сбросом сессии и редиректом.
- [ ] Реализовать перехватчик для обработки `429 Too Many Requests`, парсинг заголовка `Retry-After` и маппинг в `RateLimitException`.
- [ ] Создать и внедрить провайдер идемпотентности для генерации уникальных `Idempotency-Key` (UUIDv4) на слое Presentation.
- [ ] Интегрировать Multiplatform Settings и настроить локальное безопасное хранилище (Secure Storage) для сессий и Bearer-токена.

## Этап 2: Доменный слой и Shared Components
- [ ] Реализовать базовые абстракции MVI (интерфейсы `State`, `Intent`, `Effect` и базовый `ScreenModel`).
- [ ] Описать data-классы доменных моделей (`Client`, `ClassProgram`, `Chef`, `Slot`, `Booking`) с использованием `Integer` для денежных полей (копейки) и `UUID` для идентификаторов.
- [ ] Настроить базовую навигацию на основе Voyager для поддержки переходов между экранами (Screen) и вложенной BottomSheet-навигации.
- [ ] Настроить базовую дизайн-систему (токены цветов, типографики, отступов) и общие UI-компоненты (кнопки, снекбары, диалоги).
- [ ] Реализовать изолированный expect/actual UI-канал для отображения карт со студиями (shared component для экранов деталей слота и бронирования).

## Этап 3: Пофичевая реализация (Feature-by-Feature Checklist)

### 3.1 Фича: Авторизация (SCR-001, SCR-002)
- [ ] Создать MVI-компонент (`ScreenModel`, `State`, `Intent`, `Effect`) для логики запроса OTP, ввода кода и регистрации, включая обработку `429 Too Many Requests`.
- [ ] Сверстать UI-экраны ввода телефона и OTP (Default, Loading, Error states) на Compose.
- [ ] Интегрировать UI авторизации со слоем логики, внедрить сохранение токена в Secure Storage и настроить навигацию на главный экран.

### 3.2 Фича: Каталог слотов и расписание (SCR-003, SCR-004)
- [ ] Создать MVI-компонент для загрузки расписания, фильтрации по датам и получения детальной информации о слоте.
- [ ] Сверстать UI-экраны расписания и деталей слота (включая отображение данных шефа, цены в рублях, загрузку изображений через Coil 3).
- [ ] Реализовать продуктовые состояния для расписания: Empty state ("Нет доступных классов") и Loading state.
- [ ] Интегрировать UI со слоем логики, обработать состояние `NetworkException` (Offline) с показом снекбара и ошибку `410 Gone` при попытке открыть удаленный слот.

### 3.3 Фича: Бронирование (BS-001, BS-002, BS-003, SCR-005)
- [ ] Создать MVI-компонент для процесса бронирования: калькуляция итоговой стоимости, обработка флага аренды, генерация `Idempotency-Key`.
- [ ] Реализовать обработку бизнес-ошибок при бронировании (409 SLOT_FULL, 409 EQUIPMENT_OUT, 410 Gone, Network Timeout) с соответствующими Side Effects (алерты, снекбары).
- [ ] Сверстать UI шторки оформления брони (BS-001), экрана реквизитов с таймером оплаты (SCR-005), шторки отмены (BS-003) и выбора аллергий (BS-002).
- [ ] Интегрировать UI со слоем логики бронирования и привязать к графу навигации.

### 3.4 Фича: Профиль и активные записи (SCR-006, SCR-007, BS-004)
- [ ] Создать MVI-компонент для загрузки списка активных бронирований клиента и отправки оценки шефу.
- [ ] Сверстать UI экрана профиля (SCR-006), деталей предстоящей записи (SCR-007) и шторки оценки шефа (BS-004).
- [ ] Реализовать Empty state ("Нет активных записей") для списка бронирований.
- [ ] Интегрировать UI со слоем логики профиля, обработать успешные и ошибочные сценарии (например, таймаут при отправке рейтинга, загрузке профиля).

## Этап 4: Интеграционное тестирование и стабилизация (E2E Readiness)
- [ ] Настроить и провести сквозную проверку транзакционного сценария: Выбор слота -> Шторка подтверждения (BS-001).
- [ ] Проверить передачу параметров `slot_id`, `needs_rental_equipment` и жестко заданного `seats_count = 1`.
- [ ] Подтвердить корректную привязку `Idempotency-Key` к POST-запросу `/bookings` и его сохранение в стейте при сетевых ретраях.
- [ ] Протестировать обработку серверных конфликтов: корректное отображение UI при 409 SLOT_FULL, 409 EQUIPMENT_OUT, 410 Gone и 429 Too Many Requests.
