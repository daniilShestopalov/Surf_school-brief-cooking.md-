# Сценарии использования (Use Cases)

| ID | Название (Сценарий) | Краткое описание и шаги | Приоритет | Источник |
| :--- | :--- | :--- | :--- | :--- |
| UC-1 | Авторизация по номеру телефона | Пользователь вводит номер телефона, соглашается с обработкой ПД, получает SMS/OTP и авторизуется в системе. | Must have | `requirements_summary.md`, Раздел 2, 3 |
| UC-2 | Просмотр расписания слотов | Пользователь открывает приложение, видит расписание на 7 дней (или Empty State). При желании применяет фильтр дат. | Must have | `domain_model.md`, Раздел 3 |
| UC-3 | Бронирование слота с арендой инвентаря | Пользователь выбирает слот. Если инвентарь есть — отмечает чекбокс аренды (иначе видит заблокированный). Подтверждает бронь. Видит экран с реквизитами на оплату. | Must have | `domain_model.md`, Раздел 3; `requirements_summary.md`, Раздел 1, 3 |
| UC-4 | Заполнение медицинского профиля | Пользователь заходит в профиль и выбирает аллергии из готового списка тегов (сохраняется глобально в профиле `Client`). | Must have | `requirements_summary.md`, Раздел 1 |
| UC-5 | Отмена бронирования (Заблаговременная / Поздняя) | Пользователь нажимает "Отменить". Если отмена поздняя — подтверждает модальное окно с предупреждением о штрафе. Статус меняется. | Must have | `requirements_summary.md`, Раздел 1 |
| UC-6 | Выставление оценки шефу | После класса пользователю приходит Push. Он открывает карточку оценки, ставит от 1 до 5 звезд. Если прошло больше 24 часов — карточка пропадает. | Must have | `requirements_summary.md`, Раздел 2 |
| UC-7 | Получение уведомления об отмене студией | При отмене класса по инициативе студии, статус `Booking` меняется, пользователь получает Push с причиной отмены. | Must have | `domain_model.md`, Раздел 3 |
