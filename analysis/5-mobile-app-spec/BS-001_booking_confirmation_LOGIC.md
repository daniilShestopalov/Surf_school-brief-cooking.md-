# Логика Оформления бронирования (BS-001)

**ID:** BS-001_LOGIC  
**Тип:** Логика Bottom Sheet  
**Домен:** 04. Бронирование  
**Приоритет:** High

---

## Обзор

Логика калькуляции итоговой стоимости на клиенте, проверка наличия инвентаря (влияет на доступность чекбокса) и безопасная отправка запроса на создание брони с использованием идемпотентного ключа.

### User Story

> Как пользователь, я хочу гарантированно забронировать класс без риска двойного списания,
> чтобы быть уверенным в своей записи (US-90).

---

## Ограничения

- На текущем этапе MVP выбор количества гостей в UI не поддерживается, параметр `seats_count` жестко передается со значением 1.

## API запросы

### POST /bookings (`createBooking`)

**Триггер:** Нажатие "Подтвердить запись".

**Headers:**
| Поле | Описание | Значение |
|------|----------|----------|
| `authorization` | Bearer токен | Токен юзера |
| `Idempotency-Key` | Защита от дублей | `uuidv4()` генерируется при каждом входе в шторку или разово для текущей сессии бронирования. При ретрае по ошибке сети используется тот же ключ. |

**Параметры/Body:**
| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| `slot_id` | UUID | Слот | Входящий параметр |
| `seats_count` | Int | Места | `1` |
| `needs_rental_equipment` | Bool | Аренда | Значение чекбокса |

**Обработка ответа:**
| Результат | Действие |
|-----------|----------|
| Загрузка | Спиннер на кнопке, блокировка UI. |
| Успех (201) | Из ответа берутся `expires_at` и `id` (опционально). Открывается SCR-005, шторка BS-001 закрывается. |
| Ошибка 409 | Отображение специфичного текста в зависимости от ответа API (например, "Места закончились" или "Инвентарь занят"). |

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| 401 Unauthorized | Глобальная | Принудительный разлогин, очистка Bearer токена и перенаправление на экран авторизации SCR-001 |
| 429 Too Many Requests | Отправка `createBooking` | Системное сообщение "Слишком много запросов, пожалуйста, подождите" и блокировка кнопки отправки |
| 5xx Server Error | Отправка `createBooking` | Системный алерт "Сервис временно недоступен. Попробуйте позже" |
| NETWORK_ERR | Отправка `createBooking` | Системный алерт "Отсутствует подключение к сети". При повторной отправке запроса после сетевого сбоя (Timeout) сгенерированный ранее Idempotency-Key ДОЛЖЕН переиспользоваться и оставаться неизменным. |
