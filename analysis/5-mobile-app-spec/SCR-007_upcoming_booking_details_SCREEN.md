## Обзор

Экран с полной информацией о предстоящем классе (адрес, статус брони, наличие аренды инвентаря). Используется как для просмотра из списка, так и в качестве целевого экрана для Push-уведомлений.

### User Story

> Как пользователь, я хочу видеть полную информацию о своей брони,
> чтобы знать, куда ехать и оплачен ли класс (US-150, US-160).

## Входные данные

| Название | Тип | Описание |
|----------|-----|----------|
| `booking_id` | UUID | Идентификатор бронирования |
| `slot_id` | UUID | Идентификатор слота (может быть пустым при диплинке) |

---

## Инициализация

При переходе по диплинку (Push-уведомлению) методы `getBooking` (`GET /bookings/{id}`) и `getSlot(slot_id)` (если данные недоступны в кэше) должны быть вызваны для формирования контента.

---

## Используемые запросы

### getBooking

**Тип:** REST  
**Метод:** GET  
**Спецификация:** `analysis/api/bookings/api.yaml` → `getBooking`

**Цель:** Точечное получение информации о конкретном бронировании для корректного открытия экрана (например, из Push-уведомления).

### getSlot

**Тип:** REST  
**Метод:** GET  
**Спецификация:** `analysis/api/slots/api.yaml` → `getSlot`

**Цель:** Получить адрес, название и детали программы, если они недоступны в кэше.

## Элементы экрана

### 1. Статус-блок
| Элемент | Источник данных |
|---------|-----------------|
| Плашка статуса | Зависит от `Booking.status` ("Ожидает оплаты", "Оплачено", "Отменено") |
| Название класса | `SlotDetails.program.title` |
| Дата и время | `SlotDetails.datetime_start` |

### 2. Действия (если применимо)
| Элемент | Логика отображения | Действие |
|---------|--------------------|----------|
| Кнопка "Оплатить" | Только если `status == PENDING_PAYMENT` | Открывает SCR-005 |
| Кнопка "Отменить запись" | Если `status == ACTIVE` или `PENDING_PAYMENT` | Открывает BS-003 |

### 3. Блок деталей
| Элемент | Источник данных |
|---------|-----------------|
| Адрес (с картой) | `SlotDetails.address` |
| Доп. услуги | Если `Booking.needs_rental_equipment == true` - выводить "Включена аренда инвентаря" |
| Правила отмены | Текстовая заглушка |

## Состояния экрана

| Состояние | Условие | Отображение |
|-----------|---------|-------------|
| Loading | При инициализации из Push (Deep link) | Скелетон данных |
| Invalid (Отмена студией) | `status == CANCELLED_BY_STUDIO` | Красный баннер "Класс отменен", статус "Отменено", скрыты кнопки действий. |
| Error | Ошибка сети | Снекбар "Не удалось загрузить данные" + кнопка повтора |
