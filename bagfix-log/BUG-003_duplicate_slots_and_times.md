# [BUG-003]: Одинаковое время и дублирование слотов на разные дни

## 1. Контекст и Цель

- **Симптом или цель:** Все мастер-классы на экране расписания отображаются с одинаковым временем начала (например, 21:00) и один и тот же список повторяется при переключении на любые дни (даты).
- **Начальное состояние:** В `MockApiHandler` возвращался жестко захардкоженный список слотов с фиксированными датами в мае 2024 года. Кроме того, в `SlotsCatalogStore` список слотов, загруженный для 7-дневного окна, никак не фильтровался локально по выбранной пользователем дате (`selectedDate`), из-за чего на экране отображалось всё подряд.

## 2. Анализ Требований и Ссылки

- **Связанные требования:** Корректное отображение актуального списка мастер-классов именно на выбранную дату с правильным распределением времени.
- **API-контракты:** При запросе расписания мок должен учитывать переданные параметры (в частности `start_date`) и отдавать слоты с релевантными метками времени.

## 3. Окружение и Инструменты (Environment)

- **IDE:** Antigravity IDE (Claude Code)
- **AI-модель:** Gemini 3.1 Pro (High)
- **Дата реализации:** 2026-07-05

## 5. Реализация и Изменения

- **Выполненные действия:** 
  - **К Промту 1:** В `SlotsCatalogStore.kt` добавлена локальная фильтрация загруженного списка `domainSlots`: теперь перед маппингом в UI-модели остаются только те слоты, дата которых строго совпадает с текущим выбранным днем `state.value.selectedDate`. 
  - В `MockApiHandler.kt` добавлена обработка query-параметра `start_date`. Если он передан (или берется сегодняшний день по умолчанию), моки генерируют `datetimeStart` динамически для этого дня с шагом (10:00, 12:00, 14:00, 16:00, 18:00).
  - **К Промту 2:** Добавлены отсутствующие импорты `kotlinx.datetime.todayIn`, `kotlinx.datetime.toInstant` в `MockApiHandler.kt` и `kotlinx.datetime.toLocalDateTime` в `SlotsCatalogStore.kt` для корректной работы extension-функций.
- **Затронутые файлы:**

## Дополнительные заметки

Моковое API теперь отдает слоты, «привязанные» к запрашиваемому `start_date`. Поскольку Store делает запрос `repository.listSlots(startDate = startDate, endDate = endDate)`, где `startDate` равен выбранному дню, мок генерирует данные именно на выбранный день, и они успешно проходят новую локальную фильтрацию в Store.
