# [BUG-010]: Ошибка 400 Bad Request при открытии деталей записи (Booking Details)

## 1. Контекст и Цель

- **Симптом или цель:** При нажатии на карточку бронирования в списке записей (экран Профиля) происходило падение/ошибка: `Expected response body of the type 'class com.surfschool.features.profile.data.dto.BookingResponse ...' but was 'class io.ktor.utils.io.ByteBufferChannel ...'`. В логах: `Response status '400 Bad Request'`.
- **Начальное состояние:** В `MockApiHandler.kt` отсутствовал обработчик для запроса `GET /bookings/{id}`. Запрос попадал в ветку `else -> respondBadRequest()`, возвращая пустой ответ и статус 400. Ktor пытался распарсить пустое тело как JSON, что приводило к `NoTransformationFoundException`.

## 2. Анализ Требований и Ссылки

- **Связанные требования:** Просмотр деталей бронирования по ID.
- **API-контракты:** `ProfileApi.kt` делает вызов `httpClient.get("/bookings/$bookingId").body()`, который ожидает структуру `BookingResponse` в формате JSON (`snake_case` ключи).

## 3. Окружение и Инструменты (Environment)

- **IDE:** Antigravity IDE (Claude Code)
- **AI-модель:** Gemini 3.1 Pro (High)
- **Дата реализации:** 2026-07-06

## 5. Реализация и Изменения

- **Выполненные действия:** 
  - В `MockApiHandler.kt` добавлена новая ветка проверки `url.contains("/bookings/") && !url.endsWith("/bookings") && !url.endsWith("/rating") && method == HttpMethod.Get`.
  - В этой ветке извлекается `bookingId` из конца URL.
  - Возвращается JSON-строка, соответствующая формату `BookingResponse` (с ключами `client_id`, `slot_id`, `seats_count`, `fixed_base_price` и т.д. в `snake_case`), подставляющая извлеченный `$bookingId`.
- **Затронутые файлы:**
