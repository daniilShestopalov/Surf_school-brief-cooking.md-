# [BUG-005]: Отсутствие валидации и ограничения длины номера телефона

## 1. Контекст и Цель

- **Симптом или цель:** На экране авторизации (Шеф-стол) в поле ввода номера телефона можно было вводить любые символы (буквы, спецсимволы) и неограниченное количество цифр (например, "9999999999999999999"). Это нарушает требования к вводу стандартного российского номера телефона.
- **Начальное состояние:** В `LoginScreen.kt` коллбэк `onValueChange` в компоненте `OutlinedTextField` напрямую передавал сырой ввод (`it`) в интент `LoginIntent.PhoneChanged` без какой-либо фильтрации и ограничений. Кроме того, клавиатура не была настроена на цифровой ввод.

## 2. Анализ Требований и Ссылки

- **Связанные требования:** Ввод телефона должен принимать только цифры. Максимальная длина — 11 символов (типичный российский формат: 7XXXXXXXXXX или 8XXXXXXXXXX). 
- **API-контракты:** API для отправки кода ожидает корректный формат телефона.

## 3. Окружение и Инструменты (Environment)

- **IDE:** Antigravity IDE (Claude Code)
- **AI-модель:** Gemini 3.1 Pro (High)
- **Дата реализации:** 2026-07-06

```text
### БАГ: Некорректный ввод номера телефона (SCR-001)

**Описание:** 
На странице авторизации поле номера телефона позволяет вводить бесконечное количество символов, включая буквы и спецсимволы.

**Задача для агента:**
1. Ограничить ввод в поле `OutlinedTextField` только цифрами.
2. Ограничить максимальное количество вводимых символов длиной обычного российского номера (11 цифр).
3. Добавить подходящую настройку клавиатуры (`KeyboardType.Phone`).

**Ожидаемый результат:** 
Пользователь может ввести максимум 11 цифр, попытки ввести другие символы игнорируются, открывается цифровая клавиатура (на устройствах с экранной клавиатурой).
```

## 5. Реализация и Изменения

- **Выполненные действия:** 
  - **К Промту 1:** В файле `LoginScreen.kt` изменена обработка ввода в `onValueChange`. Теперь входящая строка фильтруется: оставляются только цифры (`filter { it.isDigit() }`), а затем результат обрезается до 11 символов (`take(11)`).
  - В `OutlinedTextField` добавлен параметр `keyboardOptions` со значением `KeyboardOptions(keyboardType = KeyboardType.Phone)` для автоматического открытия цифровой клавиатуры на мобильных устройствах.
- **Затронутые файлы:**
  - `development/client/src/desktopMain/kotlin/com/surfschool/features/auth/ui/LoginScreen.kt`
