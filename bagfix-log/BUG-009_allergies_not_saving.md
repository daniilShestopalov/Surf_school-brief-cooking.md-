# [BUG-009]: Аллергии не сохраняются и не отображаются после редактирования

## 1. Контекст и Цель

- **Симптом или цель:** После выбора аллергий в шторке (BS-002) и нажатия кнопки "Сохранить", список аллергий в профиле оставался пустым ("Нет") и не обновлялся.
- **Начальное состояние:** 
  1. В моковом сервере (`MockApiHandler.kt`) отсутствовал перехватчик для `PATCH /profile/allergies`, поэтому запрос уходил либо в пустоту (перехватывался общим `/profile`), либо игнорировался, и состояние не обновлялось.
  2. Эндпоинт `GET /profile` возвращал жестко захардкоженный ответ, где ключ для аллергий был в формате `camelCase` (`"allergyProfile": ["Орехи", "Мед"]`). Модель `ProfileResponse` ожидала `snake_case` (`@SerialName("allergy_profile")`). Из-за несовпадения ключей Ktor всегда парсил пустой список.

## 2. Анализ Требований и Ссылки

- **Связанные требования:** Корректное отображение и сохранение списка аллергий (FR-25).
- **API-контракты:** Ответ `GET /profile` должен отдавать ключи в формате `snake_case`, а обработчик `PATCH /profile/allergies` должен сохранять отправленный список в локальное состояние (state) мок-сервера.

- **IDE:** Antigravity IDE (Claude Code)
- **AI-модель:** Gemini 3.1 Pro (High)

```text
3. Добавить логирование сетевого ответа в `ProfileRepository`, чтобы убедиться, что `PATCH`-запрос завершается со статусом `200/204`, а не `4xx/5xx`.

```

## 5. Реализация и Изменения

- **Выполненные действия:** 
  - В `MockApiHandler.kt` добавлена глобальная переменная `mockAllergies` для хранения выбранных аллергий.
  - Изменен моковый ответ для `GET /profile`. Ключ `"allergyProfile"` заменен на `"allergy_profile"`, а значение формируется динамически из `mockAllergies` с помощью `joinToString`.
  - Добавлен новый обработчик для `url.endsWith("/profile/allergies") && method == HttpMethod.Patch`. Он считывает тело запроса (через `request.body.toByteArray()`), десериализует его в `UpdateAllergiesRequest` и перезаписывает `mockAllergies`.
