# [BUG-007]: Ошибка десериализации бронирований на экране профиля

## 1. Контекст и Цель

- **Симптом или цель:** При переходе на экран профиля возникает ошибка сети: `Network Error` и сообщение `JsonConvertException: Illegal input: Fields [client_id, slot_id, created_at, fixed_base_price] are required...`. Экран не может загрузить данные активных бронирований.
- **Начальное состояние:** В `MockApiHandler.kt` ответ для эндпоинта `GET /bookings` содержал ключи в формате `camelCase` (например, `clientId`, `slotId`, `createdAt`), тогда как DTO `BookingResponse` ожидал ключи в `snake_case` (согласно аннотациям `@SerialName`, например, `client_id`, `slot_id`, `created_at`). 

## 2. Анализ Требований и Ссылки

- **Связанные требования:** Корректное отображение списка бронирований в профиле пользователя (SCR-006). 
- **API-контракты:** Моковый сервер должен возвращать данные, полностью соответствующие ожидаемым Data Transfer Objects (DTO) на стороне клиента. В данном случае `BookingResponse` явно требует `snake_case` формат полей, поэтому JSON из моков должен совпадать.

## 3. Окружение и Инструменты (Environment)

- **IDE:** Antigravity IDE (Claude Code)
- **AI-модель:** Gemini 3.1 Pro (High)
- **Дата реализации:** 2026-07-06

```text
Список бронирований на экране профиля загружается без сетевых исключений, приложение успешно обрабатывает входящий JSON.
```

## 5. Реализация и Изменения

- **Выполненные действия:** 
  - Проанализирована структура `BookingResponse` в файле `ProfileDtos.kt`. Подтверждено использование аннотаций `@SerialName` с ключами в формате `snake_case`.
  - В файле `MockApiHandler.kt` для мокового ответа на `GET /bookings` ключи в формате `camelCase` заменены на требуемый `snake_case` (`client_id`, `slot_id`, `seats_count`, `created_at`, `fixed_base_price`, `equipment_tariff`, `needs_rental_equipment`).
- **Затронутые файлы:**
  - `development/client/src/desktopMain/kotlin/com/surfschool/core/network/MockApiHandler.kt`
