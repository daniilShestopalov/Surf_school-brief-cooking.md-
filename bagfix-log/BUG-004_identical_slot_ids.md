# [BUG-004]: Одинаковые ID мастер-классов в каталоге

## 1. Контекст и Цель

- **Симптом или цель:** Все мастер-классы на экране расписания отображались с одинаковым идентификатором: "Мастер-класс (ID: p0b8)".
- **Начальное состояние:** В `SlotsCatalogStore` название слота формируется искусственно из ID программы, так как бэкенд `/slots` не возвращает реальное название (там стоит комментарий `// Placeholder for program title`). Для формирования ID использовалась функция `take(4)`, которая брала первые 4 символа от `programId`. Поскольку все моковые UUID начинались с `p0b8`, UI выглядел дублирующимся.

## 2. Анализ Требований и Ссылки

- **Связанные требования:** Удобство и наглядность пользовательского интерфейса (пользователь должен визуально отличать один мастер-класс от другого).
- **API-контракты:** Формат моковых ID программы: `p0b8a211-0000-0000-0000-prog10000001`. Различающаяся часть находится в конце строки.

Слоты имеют разные идентификаторы в названиях (например, `0001`, `0002` и т.д.), что позволяет визуально их различать.

## 5. Реализация и Изменения

- **Выполненные действия:** 
  - **К Промту 1:** В файле `SlotsCatalogStore.kt` при маппинге доменной модели в `SlotItem` изменена функция `take(4)` на `takeLast(4)`. Теперь для плейсхолдера названия используются последние 4 символа `programId`, которые в моках гарантированно уникальны (0001, 0002 и т.д.).

## Дополнительные заметки

В идеале API `/slots` должно сразу отдавать хотя бы краткое название программы `programTitle`, чтобы фронтенду не приходилось собирать его из ID. Данный фикс решает визуальную проблему до реализации полноценного ответа от бэкенда.
